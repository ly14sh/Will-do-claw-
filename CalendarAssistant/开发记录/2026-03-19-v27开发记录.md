# 2026-03-19 开发记录 (v27版本)

## 需求背景

**用户报告**：课程事件点击"已完成"按钮没有效果。

**根本原因**：`AppRepository.completeScheduleEvent()` 只处理 `EventType.EVENT`，课程事件 `EventType.COURSE` 被跳过。

---

## 一、本次实现范围

### 课程事件"已结束"按钮修复

**实现要点**：
- 复用现有的 `isCompleted` 逻辑，只需 2 处修改。
- 课程事件显示"已结束"按钮，普通事件显示"已完成"按钮。
- 点击后课程事件会被标记为已完成状态。

---

## 二、关键文件变更汇总

| 文件 | 修改内容 |
|------|----------|
| `app/src/main/java/com/antgskds/calendarassistant/data/repository/AppRepository.kt` | `completeScheduleEvent()` 增加 `EventType.COURSE` 支持 |
| `app/src/main/java/com/antgskds/calendarassistant/service/capsule/CapsuleMessageComposer.kt` | 课程事件显示"已结束"按钮 |

---

## 三、修改详情

### 1. AppRepository.kt (第 458 行)

```kotlin
// 修改前
if (event != null && event.eventType == EventType.EVENT && !event.isCompleted) {

// 修改后
if (event != null && (event.eventType == EventType.EVENT || event.eventType == EventType.COURSE) && !event.isCompleted) {
```

### 2. CapsuleMessageComposer.kt (第 214 行)

```kotlin
// 修改前
CapsuleActionSpec(
    label = "已完成",
    receiverAction = EventActionReceiver.ACTION_COMPLETE_SCHEDULE
)

// 修改后
CapsuleActionSpec(
    label = if (event.eventType == EventType.COURSE) "已结束" else "已完成",
    receiverAction = EventActionReceiver.ACTION_COMPLETE_SCHEDULE
)
```

## 七、UI 优化 - 空状态样式统一

**修改时间**：2026-03-20

### 需求背景

- 实验室页的"暂无实验功能"有 Card 背景，与其他页面不统一
- 全部列表页的"暂无日程记录"显示位置靠上，不符合预期

### 修改内容

| 文件 | 修改内容 |
|------|----------|
| `ui/page_display/settings/LaboratoryPage.kt` | 去掉 Card 背景，统一为居中 Text |
| `ui/page_display/AllEventsPage.kt` | 空状态从 LazyColumn item 改为外层居中显示 |

### 修改前对比

**LaboratoryPage.kt** - 有 Card 背景：
```kotlin
Card(
    colors = CardDefaults.cardColors(containerColor = MaterialTheme.colorScheme.surfaceContainerLow)
) {
    Text("暂无实验室功能")
}
```

**AllEventsPage.kt** - 位置靠上：
```kotlin
LazyColumn {
    if (filteredEvents.isEmpty()) {
        item {
            Box(modifier = Modifier.height(400.dp)) {
                Text("暂无日程记录")
            }
        }
    }
}
```

### 修改后统一为

**LaboratoryPage.kt** & **AllEventsPage.kt**：
```kotlin
if (isEmpty) {
    Box(modifier = Modifier.fillMaxSize()) {
        Box(modifier = Modifier.align(Alignment.Center)) {
            Text(text, color = MaterialTheme.colorScheme.secondary)
        }
    }
}
```

### 效果

- 所有空状态统一使用 `secondary` 颜色
- 空状态在屏幕中央显示
- 与课表管理页样式保持一致

## 十一、UI 统一 - FAB 按钮尺寸标准化

**修改时间**：2026-03-20

### 需求背景

不同页面的 FAB 按钮根据 `uiSize` 动态调整，而主页底栏固定为 72.dp，导致视觉不统一。

### 修改内容

| 文件 | 修改内容 |
|------|----------|
| `ui/page_display/settings/TimeTableEditorScreen.kt` | FAB 尺寸统一为 72.dp / 34.dp |
| `ui/page_display/settings/CourseManagerScreen.kt` | FAB 尺寸统一为 72.dp / 34.dp |
| `ui/page_display/settings/AiSettingsPage.kt` | FAB 尺寸统一为 72.dp / 34.dp |

### 修改详情

```kotlin
// 修改前
val fabSize = when (uiSize) {
    1 -> 56.dp
    2 -> 64.dp
    else -> 72.dp
}
val fabIconSize = when (uiSize) {
    1 -> 24.dp
    2 -> 28.dp
    else -> 32.dp
}

// 修改后
val fabSize = 72.dp
val fabIconSize = 34.dp
```

### 效果

- 所有 FAB 按钮统一为 72.dp × 72.dp
- 所有 FAB 图标统一为 34.dp
- 与主页底栏保持一致

## 十三、UI 优化 - 主页搜索框样式优化

**修改时间**：2026-03-20

### 需求背景

主页搜索框样式不够理想：高度偏矮、宽度全屏、缺少关闭按钮。

### 修改内容

| 文件 | 修改内容 |
|------|----------|
| `ui/page_display/HomePage.kt` | 搜索框高度/宽度/按钮优化 |

### 修改详情

**尺寸调整**：
```kotlin
val searchBarHeight = 64.dp  // 从 56.dp 增加
```

**宽度与对齐**：
```kotlin
// 修改前
modifier = Modifier
    .padding(start = 16.dp, end = 16.dp, bottom = ...)
    .fillMaxWidth()

// 修改后
modifier = Modifier
    .padding(bottom = ...)
    .fillMaxWidth(0.75f)  // 屏幕宽度的 3/4
    .height(searchBarHeight)
```

**关闭按钮**：
```kotlin
trailingIcon = {
    IconButton(
        onClick = { isSearchMode = false },
        modifier = Modifier.padding(end = 4.dp)
    ) {
        Icon(Icons.Default.Close, ...)
    }
}
```

### 效果

| 项目 | 修改前 | 修改后 |
|------|--------|--------|
| 高度 | 56.dp | 64.dp |
| 宽度 | 全宽 | 屏幕 3/4 |
| 按钮 | 清除（有内容时） | 关闭（始终显示） |
