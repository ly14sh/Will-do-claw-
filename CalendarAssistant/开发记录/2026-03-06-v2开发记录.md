# 2026-03-06 开发记录 (v2版本)

## 新增功能

### 1. SSORS OCR 优化方案

**核心目标**：利用 ML Kit Text.Element 级别的 BoundingBox 信息，在本地通过算法将二维的视觉布局重构为一维的结构化语义文本，让 AI 更准确理解"谁在什么时候说了什么"或"哪个标签对应哪个数值"。

**实现原理**：

- **Element 级别处理**：不使用 ML Kit 的 Line 级别输出（会掩盖列间隙信息），而是直接获取最细粒度的 Element
- **噪点剔除**：过滤系统状态栏和导航栏区域的文字
- **动态行合并**：基于 Y 轴中心重叠度（50%阈值）判定同行，解决"左右分栏被识别为上下两行"问题
- **列分隔**：间距 > 2.5倍字符宽度自动插入 `|` 分隔符
- **聊天气泡识别**：根据 X 轴位置标记 `[L]`/`[R]`/`[C]`

### 3. AI 速度优化

**问题**：OCR 识别延迟过高（5-8秒）

**原因分析**：
- Prompt 要求 AI 生成 reasoning 字段，增加了不必要的思考时间
- 使用了推理模型（R1）而非通用模型（V3/Chat）

**解决方案**：
- 删除 Prompt 中的 `reasoning` 字段
- 添加"禁止思考"指令：`禁止输出任何思考过程、解释或Markdown标记。仅输出纯JSON。`
- Prompt 简化描述（"日程提取API"而非"高级日程助手"）

**预期效果**：响应时间从 5-8 秒缩短到 1-2 秒

---

### 4. 日程与取件码去重优化

**问题**：AI 可能同时返回"取件"日程和"取件码"取件事件

**解决方案**：关键信息覆盖检测

- 如果普通日程标题只包含取件关键词（如"取件"），直接删除
- 如果普通日程包含"取件+其他内容"（如"蜜雪冰城取件"），检查其他内容是否已被 pickup 事件覆盖
  - 被覆盖：删除普通日程
  - 未覆盖（如"开会后取件"）：保留两者

**修改文件**：
- `core/ai/RecognitionProcessor.kt` - 新增 `filterRedundantSchedules()` 函数
