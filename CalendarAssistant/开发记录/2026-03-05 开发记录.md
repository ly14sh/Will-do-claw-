## 修复的 Bug

### 1. 取件码动态title失效
**问题**：已取前应显示取件码，已取后/过期显示默认title，但实际始终显示默认title

**原因**：`PickupUtils.kt` 中的正则表达式要求3部分（号码|品牌|位置），而AI生成的只有2部分（号码|品牌），导致匹配失败

**修复**：`core/util/PickupUtils.kt:20`
```kotlin
// 修改前
private val PICKUP_PATTERN = Regex("【取件】([^|]+)\\|([^|]+)\|(.*)")

// 修改后
private val PICKUP_PATTERN = Regex("【取件】([^|]+)\\|([^|]+)(?:\\|(.*))?")
```

### 2. 胶囊/悬浮窗修改状态不同步到系统日历
**问题**：在APP外（通过胶囊通知或悬浮窗）修改日程状态后，状态不同步到系统日历，反而被系统日历的反向同步覆盖

**原因**：`completeScheduleEvent` 和 `checkInTransport` 方法使用 `triggerSync = false`，不会触发正向同步

**修复**：
1. `core/calendar/CalendarSyncManager.kt` - 新增单事件同步方法

   ```kotlin
   suspend fun syncEventToCalendar(event: MyEvent): Result<Unit>
   ```

2. `data/repository/AppRepository.kt` - 在状态变更后立即调用单事件同步
   - `completeScheduleEvent` 方法末尾
   - `checkInTransport` 方法末尾

## 新增功能

### AI识别 - 取件/日程独立路由

**背景**：之前的统一prompt需要AI同时理解三种场景（交通、取件、普通日程），prompt较长导致响应慢

**优化**：将取件和日程拆分为两个独立的prompt，并发请求

#### 1. 新增两个独立Prompt
- `getSchedulePrompt`：专注处理日程（general/train/taxi）
- `getPickupPrompt`：专注处理取件（pickup）

#### 2. 并发请求 + 去重逻辑
`core/ai/RecognitionProcessor.kt:113-138`

```kotlin
// 并发执行两个识别请求
val scheduleDeferred = async { analyzeSchedule(extractedText, settings) }
val pickupDeferred = async { analyzePickup(extractedText, settings) }
val scheduleEvents = scheduleDeferred.await()
val pickupEvents = pickupDeferred.await()

// 按 title+startTime 分组去重，优先保留 tag=pickup 的结果
val allEvents = scheduleEvents + pickupEvents
val finalEvents = allEvents
    .groupBy { "${it.title}|${it.startTime}" }
    .map { (_, events) ->
        events.maxByOrNull { if (it.tag == "pickup") 1 else 0 }!!
    }
```

#### 3. 日程Prompt调整
`core/ai/AiPrompts.kt:196-197`
```
如果文本中同时包含取件信息，只识别其中的日程部分，tag 设为 general。
```

## 待优化问题

### AI识别去重逻辑可能存在的问题
**场景**：如果两个完全不相关的事件碰巧时间相同，可能会被错误合并

**示例**：
- 日程 prompt：晚上6点吃饭 (general) → startTime = 今天18:00
- 取件 prompt：取件码456 (pickup) → startTime = 今天18:00
- 结果：只保留一个（pickup优先），另一个丢失

**建议优化方案**：改成分组处理
```kotlin
val pickupResults = all.filter { it.tag == "pickup" }
val scheduleResults = all.filter { it.tag in listOf("general", "train", "taxi") }
scheduleResults + pickupResults
```
