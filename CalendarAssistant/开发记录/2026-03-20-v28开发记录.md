## 八、本次追加优化（底栏水波纹特效）

### 需求背景

用户反馈底栏胶囊岛的水波纹（ripple）特效范围过大，希望水波纹与高亮胶囊大小一致。

### 原问题分析

- 原来 `clickable` 位于外层 Box（76dp × navHeight）
- 水波纹会覆盖整个可点击区域，比高亮胶囊（有 padding）范围大

### 修改内容

| 文件 | 修改内容 |
|------|----------|
| `ui/components/IntegratedFloatingBar.kt` | 将 `clickable` 从外层 Box 移至内层 Box |

### 修改详情

**`HydrogenNavIcon` 函数结构变化**：

```kotlin
// 修改前
Box(76dp × navHeight) {
    Box(padding区域) {
        if (isSelected) { 高亮背景 }
        Icon
    }
}
↓ clickable 在外层

// 修改后
Box(76dp × navHeight) {
    Box(padding区域) {
        if (isSelected) { 高亮背景 }
        Icon
    }
}
↓ clickable 在内层（有 padding 的区域）
```

### 效果

| 项目 | 修改前 | 修改后 |
|------|--------|--------|
| 水波纹范围 | 整个 76dp 区域 | 与高亮胶囊一致（有 padding） |
| 点击区域 | 整个 76dp 区域 | 与高亮胶囊一致 |
| 视觉效果 | 水波纹溢出高亮区域 | 水波纹与高亮区域精确匹配 |

### 副作用说明

点击区域同步变小（与水波纹范围一致），这是可接受的权衡。

如需实现"大点击区域 + 小水波纹"，需要使用自定义 Indication API，实现成本较高。

## 九、本次追加优化（悬浮窗滑入动画）

### 需求背景

悬浮窗的日程卡片当前使用简单的淡入淡出（fadeIn/fadeOut）动效，缺乏方向感。为提升用户体验和视觉层次感，改为更具方向性的滑入动画效果。

### 修改内容

| 组件 | 进入动画 | 退出动画 | 方向 |
|------|----------|----------|------|
| 日程列表 | 右侧滑入 + 淡入 | 右侧滑出 + 淡出 | 水平 |
| 底部输入框 | 下方滑入 + 淡入 | 下方滑出 + 淡出 | 垂直 |

### 关键文件

- `ui/floating/FloatingScheduleScreen.kt`

### 修改详情

**1. 日程列表动画（第 205-210 行）**

```kotlin
// 修改前
AnimatedVisibility(
    visible = isAppearing,
    enter = fadeIn(animationSpec = tween(enterDuration, easing = fastOutSlowIn)),
    exit = fadeOut(animationSpec = tween(exitDuration, easing = fastOutSlowIn)),
    modifier = Modifier.align(Alignment.TopEnd)
)

// 修改后
AnimatedVisibility(
    visible = isAppearing,
    enter = fadeIn(
        animationSpec = tween(enterDuration, easing = fastOutSlowIn)
    ) + slideInHorizontally(
        animationSpec = tween(enterDuration, easing = fastOutSlowIn),
        initialOffsetX = { it }  // 从屏幕右侧滑入
    ),
    exit = fadeOut(
        animationSpec = tween(exitDuration, easing = fastOutSlowIn)
    ) + slideOutHorizontally(
        animationSpec = tween(exitDuration, easing = fastOutSlowIn),
        targetOffsetX = { it }  // 向屏幕右侧滑出
    ),
    modifier = Modifier.align(Alignment.TopEnd)
)
```

**2. 底部输入框动画（第 248-253 行）**

```kotlin
// 修改前
AnimatedVisibility(
    visible = isAppearing,
    enter = fadeIn(animationSpec = tween(enterDuration, easing = fastOutSlowIn)),
    exit = fadeOut(animationSpec = tween(exitDuration, easing = fastOutSlowIn)),
    modifier = Modifier.align(Alignment.BottomCenter)
)

// 修改后
AnimatedVisibility(
    visible = isAppearing,
    enter = fadeIn(
        animationSpec = tween(enterDuration, easing = fastOutSlowIn)
    ) + slideInVertically(
        animationSpec = tween(enterDuration, easing = fastOutSlowIn),
        initialOffsetY = { it }  // 从屏幕下方滑入
    ),
    exit = fadeOut(
        animationSpec = tween(exitDuration, easing = fastOutSlowIn)
    ) + slideOutVertically(
        animationSpec = tween(exitDuration, easing = fastOutSlowIn),
        targetOffsetY = { it }  // 向屏幕下方滑出
    ),
    modifier = Modifier.align(Alignment.BottomCenter)
)
```

### 动画参数

| 参数 | 值 | 说明 |
|------|-----|------|
| `fastOutSlowIn` | `CubicBezierEasing(0.2f, 0.8f, 0.2f, 1f)` | 缓动曲线 |
| `enterDuration` | 280 ms | 进入时长 |
| `exitDuration` | 200 ms | 退出时长 |

### 技术说明

- 保留原有的 fadeIn/fadeOut，与滑入动画组合使用（`+` 操作符），效果更自然
- 使用现有的 `fastOutSlowIn` 缓动曲线，保持动画风格一致
- 所需的 `slideInHorizontally`, `slideOutHorizontally`, `slideInVertically`, `slideOutVertically` 已导入，无需额外 import
