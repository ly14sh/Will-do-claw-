# 2026-03-11 开发记录 (v15版本)

## 需求背景

本次开发主要围绕三条主线展开：

1. 修复聚合取件胶囊点击“已取”后不消失、底层取件事件不失效的问题。
2. 接入云端 Prompt 更新能力，并在发现新版本时通过对话框让用户自主决定是否拉取到本地。
3. 解耦作息表编辑逻辑，让课表不再绑定“固定 45 分钟 + 固定 12 节 + 固定晚上有课”，并支持单节课独立调整时长。

---

## 一、本次实现范围

### 1) 聚合取件胶囊修复

- 为聚合取件胶囊补上真正的“已取”动作。
- 接收器识别聚合 ID 后，改为批量完成当前活跃取件事件，而不是把聚合 ID 当普通事件 ID 处理。
- 批量筛选条件从旧的 `eventType == "temp"` 切到真实的 `pickup tag` 逻辑。

### 2) 云端 Prompt 更新能力

- 新增云端 Prompt 数据模型、本地缓存、版本比较和静默检查逻辑。
- 仅当 `remote.version > localVersion` 时弹出对话框提示用户是否更新。
- 用户点击“取消”后，本次版本不再重复打扰；只有更高版本出现时才再次提醒。
- 新增“手动检查更新”入口，方便调试和日常使用。
- 将 Cloudflare Worker 链路从原始 `workers.dev` 切换到自定义域名，解决访问超时问题。

### 3) 更新 Dialog 视觉统一

- Prompt 更新对话框样式参考“清空归档”弹窗，统一标题层级、正文字号和按钮风格。

### 4) 作息表编辑器解耦与增强

- 总节数 FAB 不再直接改单一总数，而是弹出新的三段结构弹窗。
- 早/中/晚三段节数改为独立配置，且晚上允许 `0-4` 节。
- 每节课时长不再是全局固定值，改为默认 `45` 分钟 + 单节独立覆盖。
- 点击整张课程卡片即可调整该节课时长，交互不再依赖左侧小圆点。
- 下一节课开始时间仍按“上一节结束 + 课间休息”继续推断。

## 二、关键设计决策

### 1) Prompt 更新必须用户可感知

云端 Prompt 会直接影响 OCR / 文本识别行为，不应静默覆盖本地规则。因此采用：

- 启动时后台静默检查；
- 发现新版本后弹 `AlertDialog`；
- 由用户明确选择“更新”或“取消”。

### 2) Prompt 缓存与设置解耦

Prompt 不属于普通用户设置项，而更接近“可更新规则资源”。因此没有塞进 `MySettings`，而是单独使用本地缓存保存版本与模板内容，降低设置 JSON 的耦合度。

### 3) 作息表不再依赖全局固定课时

原先作息表生成逻辑把每节课都视为 45 分钟，导致课程结构和实际学校作息不匹配。现在改为：

- 默认课时仍为 45 分钟；
- 若某节被单独修改，则以该节覆盖值为准；
- 课间与分段锚点继续负责推导后续时间。

### 4) 夜间节数允许为 0

不少课表白天上课、晚上无课，因此夜间段不应强制存在。结构弹窗把晚上范围调整为 `0-4`，既保留夜课能力，也允许无夜课课表自然建模。

## 四、验证要点

### 1) 聚合取件胶囊

- 出现聚合取件胶囊后，点击“已取”，底层取件事件应被真正完成。
- 聚合胶囊应在刷新后消失，不再残留。

### 2) Prompt 更新

- 启动 App 或手动点击“检查更新”时，若云端版本更高，应弹出 Prompt 更新对话框。
- 点击“更新”后，本地版本提升；点击“取消”后，同版本不再重复弹出。
- 在当前自定义域名下，请求链路应不再出现原来的 `workers.dev` 超时问题。

### 3) 作息表编辑

- 点击总节数 FAB，可分别调整上午/中午/晚上节数，其中晚上可选 `0`。
- 点击整张课程卡片，可弹出该节课的时长选择器。
- 修改单节课时长后，该节结束时间和后续节次开始时间应正确顺推。
- 修改课间时长后，普通后续节次时间应联动更新。
- 无夜课场景下，作息表仍能正常生成与保存。
