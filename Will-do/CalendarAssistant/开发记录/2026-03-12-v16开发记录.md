# 2026-03-12 开发记录 (v16版本)

## 需求背景

本次开发主要围绕提示词管理功能：

1. 实现提示词的导入/导出功能，方便用户备份和分享提示词。
2. 将云端 Prompt 板块从 AI 设置页面移动到数据备份页面。
3. 优化提示词导出格式，将 `\n` 转义字符转换为真实换行，提升可读性。

---

## 一、本次实现范围

### 1) 提示词导入/导出功能

- 在 `AiPrompts` 对象中新增 `exportToJson()` 方法，导出当前生效的提示词为自定义格式文本。
- 新增 `importFromJson()` 方法，支持两种格式导入：
  - 自定义格式（推荐）：人类可读的文本格式
  - JSON 格式（兼容）：保留旧版本备份文件的兼容性

### 2) 自定义导出格式设计

采用类似 ini 的自定义格式，导出后用户可直接阅读和编辑：

```
# Will do 提示词导出
# version: 2

=== userTextPrompt ===
第一行内容
第二行内容
=== end ===

=== schedulePrompt ===
...
=== end ===

=== pickupPrompt ===
...
=== end ===

=== unifiedPrompt ===
...
=== end ===
```

导入时自动检测格式：
- 如果包含 `=== userTextPrompt ===` 则按自定义格式解析
- 否则按 JSON 格式解析（兼容旧文件）

### 3) UI 调整

- **移除**：`AiSettingsPage` 中的「云端 Prompt」板块
- **新增**：`BackupSettingsPage` 中新增「提示词管理」板块
  - 标题显示提示词来源：「提示词来源：本地」或「提示词来源：云端 vX」
  - 描述：「导入/导出提示词，或检查云端更新」
  - 按钮布局：导入 | 导出（第一行），检查更新（第二行）

### 4) BackupCard 组件增强

- 新增 `extraButton` 参数，支持在原有按钮基础上添加额外按钮
- 新增 `extraButtonOnTop` 参数，控制额外按钮是否显示在顶部

---

## 二、关键设计决策

### 1) 为什么自定义格式而非 JSON

原始 JSON 格式中的字符串使用 `\n` 转义字符，用户在文本编辑器中查看时无法直观看到换行。自定义格式将 `\n` 转换为真实换行符，大大提升了可读性。

### 2) 导入兼容性

考虑到用户可能有旧版本的 JSON 格式备份，导入方法同时支持两种格式：
- 新版本导出的自定义格式
- 旧版本导出的 JSON 格式

### 3) 版本来源判断

- `version = 1`：默认提示词（本地）
- `version > 1`：云端更新过的提示词（云端）
