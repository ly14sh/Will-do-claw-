## 一、本次实现范围

### 1) 悬浮窗右滑归档功能

#### 需求说明

用户希望在悬浮窗中能够通过右滑操作快速归档日程，类似于左滑显示操作按钮的交互方式。

#### 技术实现

**修改文件**：
- `ui/floating/FloatingScheduleScreen.kt`
- `service/floating/FloatingScheduleService.kt`

**核心改动**：

1. **FloatingScheduleScreen.kt** - ScheduleCard 组件添加右滑归档逻辑：

   - 添加归档相关参数：
     ```kotlin
     val archiveTriggerPx = with(density) { 110.dp.toPx() }
     val screenWidthPx = with(density) { 400.dp.toPx() }
     ```

   - 添加右滑震动反馈：
     ```kotlin
     val isPastArchiveSwipe by remember { derivedStateOf { offsetX.value >= archiveTriggerPx } }
     var hasVibratedArchive by remember { mutableStateOf(false) }
     ```

   - 修改拖拽逻辑：
     - `onDragEnd`: 右滑超过阈值时触发归档飞出动画
     - `onHorizontalDrag`: 向右滑过阈值后阻力从 0.85 降为 0.95（鼓励飞出）

2. **FloatingScheduleService.kt** - 添加归档处理：

   ```kotlin
   "archive" -> {
       repository.archiveEvent(eventId)
       withContext(Dispatchers.Main) {
           Toast.makeText(applicationContext, "已归档", Toast.LENGTH_SHORT).show()
       }
   }
   ```

#### 交互说明

| 操作 | 效果 |
|------|------|
| 左滑 | 显示操作按钮（撤销/编辑） |
| 左滑超过阈值 | 执行撤销操作 |
| 右滑 | 拖拽卡片 |
| 右滑超过 110dp | 触发归档飞出动画 → 事件被归档 |
| 右滑震动反馈 | 阈值到达时震动 |

### 2) Flyme 控制中心按钮消失问题分析

#### 问题描述

Flyme 用户反馈：使用二合一面板（控制中心）时，实况通知被创建后，控制中心的所有按钮消失，只剩下通知内容。

#### 问题定位

**涉及文件**:
- `service/capsule/provider/FlymeCapsuleProvider.kt`

**可疑代码位置**:

1. **第 75-76 行** - 使用自定义 RemoteViews
   ```kotlin
   .setCustomContentView(remoteViews)
   .setCustomBigContentView(remoteViews)
   ```

2. **第 172-180 行** - Flyme 扩展参数
   ```kotlin
   putBoolean("is_live", true)
   putInt("notification.live.operation", 0)
   putInt("notification.live.type", 10)
   ```

#### 推测原因

当聚合取件胶囊创建时，Flyme 系统可能因 `is_live=true` + `operation=0` + `type=10` 的组合参数，将面板从"控制中心模式"（有快捷操作按钮）强制切换为"纯通知模式"（只显示通知内容），导致控制中心按钮消失。

#### 修复方案（待测试）

| 方案 | 修改内容 |
|------|---------|
| 方案1 | 将 `notification.live.operation` 从 `0` 改为 `1` |
| 方案2 | 移除 `putBoolean("is_live", true)` |
| 方案3 | 聚合胶囊不使用 Flyme 扩展参数 |
| 方案4 | 移除 `setCustomContentView` 使用默认布局 |

#### 文档输出

创建了 `docs/Flyme控制中心按钮消失问题分析.md`，记录了完整的分析过程和修复方案。
