# 2026-03-06 开发记录 (v4版本)

## 需求背景

用户反馈：在使用快速识别功能时，通知栏/控制中心无法自动收起，导致需要手动划掉才能继续操作。不同 Android 系统的实现差异较大，需要一套兼容方案。

---

## 问题分析

### 原有实现问题

1. **API 兼容性问题**：`GLOBAL_ACTION_DISMISS_NOTIFICATION_SHADE` 仅 Android 12+ 支持
2. **国产系统差异**：
   - MIUI/HyperOS：控制中心与通知栏分离，需要额外操作
   - 华为/荣耀/OPPO/vivo/魅族：包名各不相同
3. **三星/原生用户误触**：简单执行返回键会导致已收起的面板又弹出

---

## 解决方案

### 三层适配策略

| 层级 | 触发条件 | 实现方式 | 目标系统 |
|------|----------|----------|----------|
| 层级 1 | Android 12+ | `performGlobalAction(GLOBAL_ACTION_DISMISS_NOTIFICATION_SHADE)` | Pixel/三星/类原生 |
| 层级 2 | 层级 1 失败 | 系统广播 `ACTION_CLOSE_SYSTEM_DIALOGS` | 老版本/部分厂商 |
| 层级 3 | 层级 1/2 后 180ms | 检测包名后决定是否执行 Back 补刀 | 国产通控分离 ROM |

### 层级 3 智能判断逻辑

```kotlin
// 检测当前顶层窗口包名
val systemUiPackages = setOf(
    "com.android.systemui",    // 通用/原生/MIUI/OneUI
    "com.coloros.systemui",   // OPPO/ColorOS
    "com.oppo.systemui",      // OPPO 旧版
    "com.vivo.systemui",      // vivo/OriginOS
    "com.huawei.systemui",    // 华为/EMUI
    "com.hihonor.systemui",   // 荣耀/MagicUI
    "com.meizu.systemui"      // 魅族/Flyme
)

// 只有当前窗口仍是系统 UI 时才执行 Back 补刀
if (systemUiPackages.any { currentPackage.contains(it) }) {
    performGlobalAction(GLOBAL_ACTION_BACK)
}
```

### 关键参数

- **延时时间**：180ms（避开部分设备 150ms 动画临界态）
- **权限说明**：`performGlobalAction` 在无障碍服务中无需 `WRITE_SECURE_SETTINGS`

## 通知文案优化

### 1. 普通通知文案优化

**需求**：消除标题和内容的重复信息，让通知更简洁易读。

**问题**：原 ContentText 强行拼接 title，导致信息重复。

**解决方案**：智能文案生成，格式为：
- 提醒倒计时（倒计时类）→ 前缀（如 "[10分钟后]"）
- 时间段 → 核心（如 "15:04 - 16:00"）
- 地点 → 【地点名】
- 行动建议 → 根据 Tag（取件/检票/用车等）

**示例**：
- 原："取件码: 顺丰快递到件"
- 新："[10分钟后] 15:04 【湖北工业大学红房子服务站】请前往取件"

**增强**：
- 使用 BigTextStyle 防止长文本被截断
- 时间格式防御性处理（截取 HH:mm）

**修改文件**：

| 文件 | 修改内容 |
|------|----------|
| `service/receiver/AlarmReceiver.kt` | companion object 添加 showStandardNotification 方法，集成智能文案生成 |

### 2. 修复即时日程不提醒

**问题**：识别取件码生成的日程时间是"当前时间"，导致代码执行到设置闹钟时，时间已变成"过去"，被 `if (time > now)` 逻辑过滤掉，用户收不到通知。

**解决方案**：在调度器中增加"过去但 < 1分钟"的宽容判断。

**核心逻辑**：
```kotlin
val now = System.currentTimeMillis()
val immediateThreshold = 60 * 1000L  // 1分钟阈值

if (triggerTime > now) {
    // 未来时间：正常设置系统闹钟
} else if (now - triggerTime < immediateThreshold) {
    // 过去但 < 1分钟：立即发送通知（容忍 OCR 处理耗时）
    AlarmReceiver.showStandardNotification(context, event, label)
} else {
    // 很久以前的时间：忽略（避免历史日程刷屏）
}
```

**优点**：
- 不仅解决取件码问题，任何即时创建的日程都能收到通知
- 1分钟阈值确保手机重启时不会把旧日程全部弹出
- 增强了整个应用通知系统的健壮性

**修改文件**：

| 文件 | 修改内容 |
|------|----------|
| `service/receiver/AlarmReceiver.kt` | companion object 添加 showStandardNotification 方法（internal），支持直接调用 |
| `service/notification/NotificationScheduler.kt` | scheduleReminders 方法添加 3 处即时通知逻辑（用户自定义提醒、全局提前提醒、日程开始通知） |
