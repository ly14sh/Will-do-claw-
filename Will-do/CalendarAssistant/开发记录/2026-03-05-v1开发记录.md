# 2026-03-05 开发记录 (v1版本)

## 新增功能

### 1. 国产系统通知中心收起适配

**问题**：在国产系统（ColorOS/FlymeOS/MIUI/HyperOS/OriginOS）上，点击快捷设置磁贴后，控制中心/通知中心不会自动收起，导致截图内容包含通知栏，影响识别效果

**原因**：
- 国产系统采用"通控分离"设计（通知栏和控制中心分开）
- 标准 API `GLOBAL_ACTION_DISMISS_NOTIFICATION_SHADE` 在部分 ROM 上只能收起通知栏，收不起控制中心

**解决方案**：采用"组合拳 + 延时"策略，层层补刀确保收起成功

**修改文件**：`service/accessibility/TextAccessibilityService.kt`

```kotlin
fun closeNotificationPanel(): Boolean {
    // 第1层：API 12+ 专用 API（针对 Pixel/三星/类原生）
    if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.S) {
        performGlobalAction(GLOBAL_ACTION_DISMISS_NOTIFICATION_SHADE)
    }

    // 第2层：发送系统广播（针对老版本系统或华为/魅族）
    try {
        sendBroadcast(Intent(Intent.ACTION_CLOSE_SYSTEM_DIALOGS))
    } catch (e: Exception) {
        Log.w(TAG, "发送系统广播失败: ${e.message}")
    }

    // 第3层：模拟返回键 + 150ms 延时（针对 OPPO/vivo/小米 HyperOS 的终极杀招）
    Handler(Looper.getMainLooper()).postDelayed({
        performGlobalAction(GLOBAL_ACTION_BACK)
    }, 150)

    return true
}
```

**兼容性**：

| 系统 | 方案层级 | 说明 |
|------|---------|------|
| Pixel/类原生 | 第1层 | API 12+ 直接生效 |
| 华为/EMUI | 第2层 | 系统广播生效 |
| 魅族/Flyme | 第2层 | 系统广播生效 |
| OPPO/ColorOS | 第3层 | 返回键生效 |
| vivo/OriginOS | 第3层 | 返回键生效 |
| 小米/MIUI/HyperOS | 第3层 | 返回键生效 |

**核心原理**：
- Android CDD 规定：任何模态面板覆盖在屏幕最上层时，按下"返回键"必须关闭它
- 这是 Android 最底层的交互逻辑，没有厂商敢去掉，因此 GLOBAL_ACTION_BACK 是真正的"万能钥匙"
